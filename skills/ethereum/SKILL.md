# ethereum — ship production EVM code

Opinionated, security-first EVM development. **Verify chain facts live — don't trust training data**; networks, gas, and standards move fast. (Distilled in our own way from Austin Griffith's `ethskills` — ethskills.com. Current-network facts + agent-era standards live in [`references/reference.md`](references/reference.md).)

## Architecture (start here)
- **Fewest contracts that work.** Most dApps need **0–2 contracts, not 5–10; three is the MVP ceiling.** Solidity is for ownership, transfers, and commitments — not a database or backend.
- **Every function needs a caller who pays gas.** Audit "what if nobody calls it?"
- **Inherit, don't reinvent.** OpenZeppelin for ERC-20/721/1155/4626, access control, and math. Rolling your own is the bug.
- **Pick a chain for its superpower**, not because "mainnet is expensive." Don't deploy to 5 chains on day one. Anti-patterns: one god-contract, needless factories, storing profiles/images onchain.

## Security (the checklist)
- **Checks-Effects-Interactions**, with OZ `ReentrancyGuard` as a backstop.
- **Access control:** OZ `Ownable`/`AccessControl`; transfer ownership to a **Gnosis Safe multisig**, never leave a single EOA.
- **Token decimals:** USDC/USDT are **6 decimals, not 18** (the #1 "where did my money go" bug). Never hardcode `1e18` — read `IERC20Metadata(token).decimals()`.
- **`SafeERC20`** (`safeTransfer`) always — USDT doesn't return a bool.
- **Math:** multiply before divide (division truncates to zero); percentages in basis points `(amount * FEE_BPS) / 10_000`.
- **Oracles:** never a DEX spot price (flash-loan manipulable). Chainlink with a staleness check `require(block.timestamp - updatedAt < 3600)`, or a Uniswap V3 TWAP.
- **MEV:** never `amountOutMinimum: 0`; set 0.5–1% slippage; use Flashbots Protect RPC (`rpc.flashbots.net`).
- **No `delegatecall` to user-supplied addresses.** No infinite approvals (`type(uint256).max`) — approve exact/bounded.
- **Proxies:** UUPS over Transparent; `initializer` not constructor; `_disableInitializers()`; append-only storage.
- **ERC-4626 first-depositor inflation** → virtual offset (OZ v5 default).
- **Before deploy:** `slither .`, `mythril`, `forge test --fuzz-runs 10000`, then `forge verify-contract`. Run the audit in a **separate agent / fresh context** to avoid self-review bias.

## Testing (Foundry)
- Foundry is primary: `forge test` (256 fuzz runs default), `--fuzz-runs 10000`, `--fork-url <RPC>`, `--fork-block-number`, `forge snapshot` (gas), `-vvv`.
- Foundry auto-fuzzes parameterized tests — constrain with **`bound(x, min, max)`, not `vm.assume()`**.
- `foundry.toml`: `[fuzz] runs=1000` min for math; `[invariant] runs=512 depth=50` with a Handler contract.
- Cheatcodes: `vm.prank()`, `vm.expectRevert()`, `vm.expectEmit()`, `vm.createSelectFork`.
- **Fork-test all external protocol calls** (Uniswap/Aave/Chainlink) and non-standard tokens (USDT / fee-on-transfer). Test zero/max/empty edges and economic invariants (e.g. total deposits == balance). Don't test getters, language features, or OZ internals.
- (Hardhat 3 — TS-first with Solidity testing + fuzzing — is a valid alternative.)

## dApp frontend (Scaffold-ETH 2)
- Scaffold with **`npx create-eth@latest`** — never `forge init` or a hand-rolled Next.js.
- In dev, **always fork**: `yarn fork --network <chain>` (not `yarn chain`); keep `targetNetworks: [chains.foundry]` (31337) while forking.
- Use **Scaffold hooks, not raw wagmi:** `useScaffoldReadContract` (`watch: true`), `useScaffoldWriteContract` + `writeContractAsync({ onBlockConfirmation })`, `useScaffoldEventHistory`.
- `deployedContracts.ts` is auto-generated by `yarn deploy` — never edit; third-party ABIs go in `externalContracts.ts`. RainbowKit is built-in — don't add wallet logic.
- **Four-state single button:** Connect → Switch Network (first) → Approve → Execute. Never show Approve + Execute at once.
- Display with `formatUnits(amount, decimals)`, submit with `parseUnits` — never raw base units; give USD context. Per-action loading flags (`isApproving`, `isStaking`), never one shared `isLoading` (`isPending` clears on tx-hash, not confirmation — add a cooldown + refetch to stop double-submits). Parse errors into plain language.

## Secrets (hard rule)
**Never commit private keys or API keys** — bots exploit leaked secrets in seconds, and AI agents are the #1 source of leaked credentials. Use `.env` + `$RPC` placeholders; never fabricate a contract address (verify onchain).
